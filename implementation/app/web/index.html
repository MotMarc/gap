                Independent ecosystem governance

                
Cryptographic identity does not equal ecosystem trust.


                

                    Provider identity documents prove which signing keys belong to
                    a provider. The GAP Trust Registry separately records whether
                    that provider is currently approved to participate.
                

            


            

                

                    Decision model
                    
Two independent questions


                    

                        

                            Cryptographic validity
                            
                                Is the credential authentic under the provider's
                                published key history?
                            
                        


                        +

                        

                            Registry trust
                            
                                Is the provider currently approved by the independent
                                GAP registry?
                            
                        


                        =

                        

                            Overall GAP validity
                            
                                Both conditions must be true at verification time.
                            
                        

                    

                


                

                    Public status model
                    
Provider lifecycle


                    

                        
                            Self-declared
                        
                        
                            Applicant
                        
                        
                            Approved
                        
                        
                            Suspended
                        
                        
                            Removed
                        
                    


                    

                        Only approved providers may issue new credentials. A
                        credential from a suspended or removed provider can remain
                        cryptographically authentic while failing overall trust.

                How GAP works

                
Public provenance with controlled private attribution.


                

                    GAP separates public artifact verification from private
                    identity retention and lawful disclosure.
                

            


            

                

                    1
                    
Generation request

                    

                        A provider receives a prompt and private account
                        reference.
                    

                


                
→


                

                    2
                    
Artifact generation

                    

                        The provider generates the artifact and calculates its
                        SHA-256 digest.
                    

                


                
→


                

                    3
                    
Credential issuance

                    

                        A public Generation Credential is signed using the
                        provider's Ed25519 key.
                    

                


                
→


                

                    4
                    
Independent verification

                    

                        Anyone can verify the file hash, credential signature,
                        provider identity and independent registry trust.
                    

                


                
→


                

                    5
                    
Controlled disclosure

                    

                        Private attribution is disclosed only after structured
                        authorisation checks and audit logging.
                    

                

            


            

                

                    

                        Participating trust domains
                        
Multi-provider ecosystem

                    


                    
                        Discovering providers
                    
                


                

                    Every participating provider publishes an independent identity
                    document and complete verification-key history. The GAP Trust
                    Registry separately decides whether that provider is approved.
                    A credential cannot be reassigned to another trust domain, and a
                    revoked key cannot be silently reused.
                


                

            


            

                

                    

                        Cryptographic continuity
                        
Signing-key lifecycle

                    

                


                

                    GAP separates permission to issue new credentials from the
                    ability to verify historical credentials. Identity documents
                    retain lifecycle state so verifiers can make the correct
                    decision for every referenced key ID.
                


                

                    

                        Active
                        
Issue and verify

                        

                            The provider's single active key signs new credentials
                            and validates credentials already issued with that key.
                        

                    


                    

                        Retired
                        
Verify history only

                        

                            A retired key cannot issue new credentials, but authentic
                            historical credentials remain verifiable.
                        

                    


                    

                        Revoked
                        
Reject explicitly

                        

                            A revoked key stays published with revocation metadata so
                            every credential that references it is rejected.
                        

                    

                

            


            

                

                    Public
                    
Generation Credential

                    

                        Provider, model, generation ID, artifact digest and
                        digital signature.
                    

                


                

                    Private
                    
Attribution Record

                    

                        Account reference and prompt hash retained by the
                        provider.
                    

                


                

                    
                        Controlled
                    
                    
Disclosure

                    

                        Authorisation validation, retention enforcement and an
                        auditable disclosure decision.
